# Spec: send-message

## 한 줄 정의
사용자가 Send 버튼을 클릭하면 입력 텍스트가 사용자 메시지로 전송되고, 활성 캐릭터의 응답 플레이스홀더가 추가되며, 작성기가 비워진다.

## 진입점
- 버튼: 하단 메시지 작성기의 Send 버튼 클릭

## 사용자 흐름
1. 사용자가 작성기에 텍스트를 입력한다.
2. Send 버튼을 클릭한다.
3. 입력 텍스트가 사용자 메시지로 채팅에 추가된다.
4. 활성 캐릭터의 응답 플레이스홀더가 추가된다.
5. 작성기 입력 필드가 비워진다.

## 성공 조건
- [ ] 비어있지 않은 입력으로 Send 클릭 시 사용자 메시지와 캐릭터 응답 플레이스홀더가 추가된다.
- [ ] 입력 필드가 비워진다.

## 실패 / 취소 흐름
- 공백만 있는 입력으로 Send 클릭 → 아무것도 전송되지 않는다.
- Engine 연동이 대기 중인 경우 → UI가 차단되지 않고 플레이스홀더 응답이 표시된다.

## 경계 / 예외
- 다른 캐릭터 선택 후 Send → 응답 화자가 선택된 캐릭터로 설정된다.
- 빈 입력으로 Send → 빈 메시지가 생성되지 않는다.

## 범위 외
- Enter 단축키 전송 동작.
- Engine 응답 생성 세부사항.
