# Spec: modal-done

## 한 줄 정의
사용자가 모달의 Done 버튼을 클릭하면 임시 변경 사항이 검증 및 커밋되고, 모달이 닫히며, 관련 UI 패널이 갱신된다.

## 진입점
- 버튼: 모달 하단의 Done 버튼 클릭

## 사용자 흐름
1. 사용자가 모달에서 필드를 편집한다.
2. Done 버튼을 클릭한다.
3. 입력 데이터가 검증된다.
4. 유효한 경우 데이터가 저장되고 모달이 닫히며 관련 패널이 갱신된다.

## 성공 조건
- [ ] 유효한 데이터로 Done 클릭 시 데이터가 저장되고 모달이 닫힌다.
- [ ] 캐릭터 에디터에서 Done → 캐릭터가 추가/갱신된다.
- [ ] 페르소나 에디터에서 Done → 오른쪽 패널 페르소나 값이 갱신된다.
- [ ] 설정에서 Done → 설정이 저장되고 UI에 즉시 반영된다.

## 실패 / 취소 흐름
- 필수 필드가 유효하지 않은 상태에서 Done → 모달이 열린 상태로 유지되고 실행 가능한 유효성 메시지가 표시된다.

## 경계 / 예외
- 모달 유형별로 검증 규칙이 다를 수 있다.

## 범위 외
- 각 모달 유형별 검증 규칙 세부사항.
- 데이터 영속화 메커니즘.
