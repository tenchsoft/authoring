# Spec: story-format-roundtrip

## 한 줄 정의
사용자가 Story 프로젝트를 저장하고 다시 열면 원고, 캐릭터, 월드 빌딩, 설정 등 모든 데이터가 손실 없이 복원된다.

## 진입점
- 자동: 편집 중 Ctrl+S 또는 자동 저장으로 프로젝트 파일이 기록된다.
- 메뉴: File > Save / Save As / Open으로 프로젝트 파일을 저장/불러온다.

## 사용자 흐름
1. 사용자가 원고, 캐릭터 시트, 월드 빌딩 항목, 타임라인 이벤트, 용어집 항목 등을 편집한다.
2. 저장 시 모든 데이터가 로컬 프로젝트 파일로 기록된다.
3. 사용자가 앱을 닫았다가 다시 연다.
4. File > Open으로 프로젝트를 열면 이전 편집 상태가 그대로 복원된다.
5. 캐릭터 관계도, 마인드맵 레이아웃, 사이드바 탭 위치 등 UI 상태도 유지된다.

## 성공 조건 (Acceptance Criteria)
- [ ] 프로젝트 파일을 열면 모든 챕터 내용이 그대로 복원된다.
- [ ] 캐릭터 이름, 설명, 관계, 이미지가 손실 없이 복원된다.
- [ ] 월드 빌딩 항목(장소, 규칙, 역사)이 그대로 복원된다.
- [ ] 타임라인 이벤트 순서와 날짜가 유지된다.
- [ ] 용어집 항목과 참조가 유지된다.
- [ ] 마지막으로 편집한 챕터/섹션으로 스크롤 위치가 복원된다.
- [ ] 저장 중 오류가 발생하면 사용자에게 에러 메시지가 표시되고 이전 상태가 보존된다.
- [ ] 손상된 파일을 열면 에러 메시지와 함께 복구 시도 안내가 표시된다.

## 실패 / 취소 흐름
- 디스크 공간이 부족하면 저장 실패 메시지가 표시된다.
- 파일이 다른 프로세스에 의해 잠겨 있으면 읽기 전용으로 열린다.
- 저장 중 앱이 강제 종료되면 다음 실행 시 자동 복구가 제안된다.

## 경계 / 예외
- 매우 큰 프로젝트(100+ 챕터, 200+ 캐릭터)도 합리적인 시간 내에 열리고 저장된다.
- 동시에 두 창에서 같은 프로젝트를 열면 충돌 경고가 표시된다.
- 파일을 수동으로 편집하여 포맷이 손상된 경우 에러 처리된다.

## 범위 외
- 클라우드 동기화는 별도 spec으로 분리.
- 내보내기(TXT, Markdown, EPUB)는 별도 spec으로 분리.
